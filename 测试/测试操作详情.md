<!-- -->

1.  新增实时轮询接口：

    **新增后端路由**：增加了一个全新的接口
    \@app.get(\"/api/poll_recommend\")。

    **游标对比机制**：这个接口会读取用户的
    last_seen_proj_id，去数据库里查有没有比这个 ID
    更大的新项目，如果有，就把游标更新，并把新项目返回给前端。

    **算法升级（模糊匹配+忽略大小写）**：在匹配算法上，把你原本的集合交集判断（my_skills_set.intersection）彻底废弃了。改为了：

    把用户的技能和项目的标签全都加上 .lower()（消除大小写差异）。

    把用户的技能存为列表（List），使用 if any(skill in proj_tags_str for
    skill in
    my_skills):。这意味着只要项目标签的字符串里"包含"了你的某个技能（比如项目标签是"Java后端"，我们填"Java"），就会被成功匹配。

2.  大厅静态渲染逻辑的同步：

    **修改 get_dashboard_panels 函数**：

    开头的 my_skills_set 改成了 my_skills 列表，并全部 .lower()。

    底部的打分逻辑（score 计算）改成了 score = sum(1 for skill in
    my_skills if skill in
    proj_tags_str)。这样只要是子串包含，就会给项目加分，并把它推到右侧面板的前
    5 名。

3.  前端 JavaScript 的改造

    为了配合后端的实时推送，我对 HTML 里嵌着的 GLOBAL_JS 字符串动了刀：

    **接入推荐轮询**：在底部的 setInterval (每 2.5 秒执行一次的定时器)
    里，加了一段 fetch(\'/api/poll_recommend\') 的逻辑。

    **触发弹窗与重绘**：当前端收到后端传来的新推荐项目时，会立刻调用
    showToast() 弹出一个蓝色的系统提示框，并调用 refreshDashboard()
    局部刷新大厅面板，把新项目刷出来。

    **移除隐患代码**：删掉了之前存在浏览器 localStorage
    里的本地推荐游标变量，彻底改为由后端数据库控制，保证 A/B
    账号测试时绝对稳定。
