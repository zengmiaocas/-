1. users (用户信息表)
记录平台所有注册用户的详细资料。

phone (TEXT): 主键，用户的手机号，也是登录账号。

name (TEXT): 用户的真实姓名或昵称。

password (TEXT): 经 SHA-256 哈希加密后的密码。

session_id (TEXT): 预留字段，目前通过独立的 user_sessions 表管理会话。

college, major, class_name (TEXT): 学院、专业、班级信息。

qq, wechat (TEXT): 联系方式。

bio (TEXT): 个人简介/个性签名。

skills (TEXT): 个人技能标签（如 "Python, UI设计"），用于推荐匹配。

honors (TEXT): 个人荣誉奖项。

student_id (TEXT): 学号（要求12位数字），可用作备用登录名。

is_first_login (INTEGER): 是否首次登录（1-是，0-否），用于控制强制完善资料的弹窗。

created_at (TIMESTAMP): 注册时间。

2. projects (组队项目表)
记录队长发布的招募信息。

id (INTEGER): 自增主键，项目唯一标识。

title (TEXT): 项目标题（如 "蓝桥杯寻找后端开发"）。

leader_phone (TEXT): 队长手机号，关联 users 表。

description (TEXT): 项目详情描述，支持 Markdown 格式。

tags (TEXT): 项目技术要求或业务标签（如 "Java, 算法"）。

base_members (INTEGER): 建队时已有的初始人数。

required_members (INTEGER): 项目期望的总人数（满员后自动截止）。

status (TEXT): 当前状态（'招募中' 或 '已截止'）。

is_deleted (INTEGER): 软删除标记（1-已删除，0-正常）。

is_hidden (INTEGER): 是否仅自己可见（1-隐藏，0-公开）。

3. applications (申请记录表)
记录队员向项目发起的申请及队长审批状态。

id (INTEGER): 自增主键。

proj_id (INTEGER): 申请的目标项目 ID。

applicant_phone (TEXT): 申请人的手机号。

status (TEXT): 申请状态（'待审核', '已同意', '已拒绝', '已取消', '已移出'）。

applicant_visible (INTEGER): 申请人是否在面板中可见此记录（用于前端“删除记录”操作，属于软删除）。

leader_visible (INTEGER): 队长是否在面板中可见此记录（软删除）。

leader_read / applicant_read (INTEGER): 用于控制前端红点角标（1-已读，0-未读）。

4. messages (消息内容表)
存储私聊和群聊的所有聊天文本。

id (INTEGER): 消息的自增 ID。

sender_phone (TEXT): 发送方手机号（如果是系统消息则为 'system'）。

chat_type (TEXT): 聊天类型（'private' 表示私聊，'group' 表示群聊）。

target_id (TEXT): 消息接收对象。如果是群聊，存的是 proj_id；如果是私聊，存的是对方的 phone。

content (TEXT): 消息文本内容。

5. chat_state (聊天状态维护表)
核心表：用于维护每个用户在不同会话中的读取进度和删除操作。

phone (TEXT): 当前操作用户的手机号。

chat_type (TEXT): 会话类型（'private' 或 'group'）。

target_id (TEXT): 会话目标 ID。

last_read_msg_id (INTEGER): 该用户在此会话中最新读到的消息 ID（用于计算未读红点数：总消息数减去此 ID 后的消息）。

cleared_up_to_msg_id (INTEGER): 该用户清空聊天记录锚点。此 ID 之前的消息对该用户不再显示（但不删除库里的原始数据）。
